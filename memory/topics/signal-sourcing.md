# Signal sourcing (Reddit-first)

How Lily finds candidate signals to write into `Brainstorm/Signals/_raw/YYYY-MM-DD/`.

## Scope
- Prioritize **productivity / to-do lists / workflow** signals.
- De-emphasize **AI agents** content unless it’s directly about practical workflow automation that ordinary users actually adopt.
- Include **general life friction** signals too (fitness, cooking, health habits, etc.), not just software/business.
- Still prefer signals that can become a clear **Problem** and optionally an **Idea**.

## Subreddit strategy
- Maintain a **core list** (high signal-to-noise) plus a small **rotating satellite list**.
- Bias the core list toward **productivity/to-do/workflow** communities (incl. ADHD/productivity).
- Keep AI-agent-heavy subs as **satellites** only, unless they repeatedly produce workflow-adoption signals.
- Periodically prune subreddits that rarely produce promotable signals.

## Feed mix (efficiency)
Do **not** rely on New-only.
- Use **Top (week)** for validated pain / strong engagement.
- Add a **limited New** slice for freshness / emerging problems.
- Score candidates and pick the best set overall.

## Report composition targets (hard bias)
For each daily Top 10 report:
- **6–8** signals should be **productivity / to-do / workflow**.
- **1–3** signals can be **general life friction** (fitness/cooking/health habits).
- **0–1** signals can be **AI agents**, and only if it is clearly about practical workflow adoption (not hype).

## Heuristic scoring (high level)
Prefer posts with:
- Clear pain + context + constraints (time/budget/tools/skill/workflow)
- Evidence of demand (meaningful comments, repeats, strong reactions)
- Concrete job-to-be-done (“I’m trying to X but Y”)

Avoid/skip:
- Memes/drama, low-context opinion prompts, pure link dumps

## Data hygiene
- Dedupe across subreddits/feeds.
- Use canonical permalink from Reddit API.
- Verify the post subreddit matches the expected feed (avoid redirects/mislinks).
